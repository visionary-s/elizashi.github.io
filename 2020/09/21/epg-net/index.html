Overview

The EPG supports GPRS, EPS, and CUPS-based architecture technologies.

  • GPRS: for GSM and WCDMA systems
  • EPS: for LTE system and the non-3GPP networks

CUPS

EPG uses a Control and User Plane separation (CUPS) architecture.
A CP can be connected to one or several UP nodes, also a UP can be connected to one or several CP nodes.
The CPl and the UP communicate through the Sxb interface for PGW-C and PGW-U, and the Sxa interface for SGW-C and SGW-U, using the Packet Forwarding Control Plane (PFCP) protocol.

EPC Network Functions and Interfaces with CUPS
.

EPG can act as one or more of the following network functions:

func role
PGW-C session connectivity control for the PCEF, includes selection and control of PGW-U with respect to its functions
SGW-C session connectivity control, includes selection and control of the SGW-U with respect to its functions
PGW-U UP connectivity towards external PDNs. It is controlled by the PGW-C function
SGW-U UP connectivity between radio access network and packet gateway. It is controlled by the SGW-C function
