Architecture

Architecture mainly consists of two parts:

  • Application: software
  • Platform: IPOS, Hardware

Platform

logical architecture of the SSR platform

The key components of the platform logical architecture are as follows:

  • Route-Processor/Switch (RPSW): RP = Route-Processor, main controller in system. SW = switch = SW card
  • SW: SW card
  • Alarm/Switch (ALSW)
  • Line Card: connect with external nodes and provide Layer 2 & 3 with data forwarding
  • Platform Control Plane Switching and Switch Fabric: A dedicated 10-Gigabit Ethernet Control Plane switching and a switch fabric manage communication between controller cards (PRSW, ALSW, SW) and line cards
  • Smart Services Card (SSC): for details see section Hardware

Application

Appliction = CP + UP apps running on the SSCs. Each SSC is assigned as Control Processing Board (CPB) or Packet Processing Board (PPB).

  1. Control plane app

    CP app function & role others
    GSC (Global Session Controller) 1. admission control
    2. shared IP pool functionality    		 can run on any CPB
    PSC (PGW Session Controller) 1. establishes and controls user sessions towards the APN networks & external nodes (e.g. Online Charging System (OCS)) by processing GTP-C messages received on the Gn/Gp, S5/S8-C, GTP-based S2a, and S2b interfaces
    2. gathering of performance statistics, charging data generation, forwarding of PGW Charging Data Records (CDRs) and Rf ACRs    		 All PSC instances share a single IP address per network and the PGW-C communicates each logical interface IP address to the surrounding nodes in the same network through signalling messages
    SSC (SGW Session Controller) 1. establishes and controls user sessions by processing GTP-C messages received on the S4-C, S5/S8-C, and S11-C interfaces
    2. charging data generation for SGW-CDRs and Rf ACRs    		 Same single IP mode as above

    For details about control plane functionality, see section PGW-C overview and SGW-C overview

  2. User plane app
    runs on user plane PPBs

    UP app function & role others
    Data plane handling real-time user packet
    PFCP End-point control plane of the UP
    the end-point of PFCP (Packet Forwarding Control Plane) communication between the Sx and N4 interface
    session mgrmt    		 stateless
    Timer Wheel timer created for session
    check DB for session expire    		 stateless
    DB store session info

Deployment Types

  1. EPG runs as a standalone UP node

    Above shows an example of a standalone UP node acting as a combined PGW-U and SGW-U, with all its network IP addresses and ranges.

  2. EPG runs with the UP node functionality and the CP node functionality deployed on the same SSR chassis, where CP and UP share the same O&M admin login address but are logically two separate single nodes.

    Above shows an example of an EPG running as a single node with all its applications and network IP addresses and ranges.
    The minimum configuration for a single node deployment:

    • Two CPBs: the PGW-C runs the PSC application and the SGW-C runs the SGW Session Controller app. If the node acts as a PGW-C, one CPB also runs the GSC application.
    • Two PPBs running the UP app, which can be used as an SGW-U, a PGW-U or as a combined SGW-U and PGW-U
    • Two RPSWs
